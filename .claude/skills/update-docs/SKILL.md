# docs/info.md 업데이트

아래 단계를 순서대로 수행하세요.

## 1. 현재 구조 스캔

다음 항목의 실제 파일/폴더 구조를 확인합니다:

- **Frontend App Router**: `frontend/src/app/` — Route Group과 페이지 목록
- **Frontend 컴포넌트**: `frontend/src/components/` — 하위 폴더 목록
- **Backend 라우터**: `backend/src/routers/` — 파일 목록
- **Backend 에이전트**: `backend/src/agents/` — 파일 목록
- **최상위 구조**: 프로젝트 루트의 주요 폴더/파일

## 2. docs/info.md 현재 내용 읽기

`docs/info.md`를 읽어 현재 문서화된 구조를 파악합니다.

## 3. 변경점 비교

실제 구조와 문서를 비교하여 다음을 식별합니다:
- 새로 추가된 폴더/파일
- 삭제된 폴더/파일
- 이름이 변경된 폴더/파일
- 역할이 바뀐 항목

## 4. docs/info.md 업데이트

변경된 항목만 선택적으로 수정합니다. 수정 대상 섹션:

- **섹션 2** (`최상위 폴더/파일 구조`) — 루트 레벨 변경 시
- **섹션 3** (`Frontend/Backend 폴더 구조`) — 소스 폴더 구조 변경 시
- **섹션 4** (`Claude Code 커맨드 스킬`) — 새 스킬 추가 시

수정 시 주의사항:
- 역할 설명은 기존 톤앤매너를 유지 (한국어, 간결하게)
- 삭제된 항목은 문서에서도 제거
- 새 항목은 적절한 위치에 추가
- 설명이 불명확하면 해당 파일을 직접 읽고 역할을 파악

## 5. 변경 요약 보고

업데이트된 내용을 사용자에게 간결하게 보고합니다:
- 어떤 섹션이 변경되었는지
- 추가/삭제/수정된 항목 목록
